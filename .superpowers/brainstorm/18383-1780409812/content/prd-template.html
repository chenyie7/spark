PRD 模板：PM Agent 最终交付物长什么样？

PRD 的结构决定了 Agent 在对话阶段要收集哪些信息

候选模板

A

精简版（适合快速迭代）

1. 背景与目标（1-2段）
2. 用户故事（As a... I want... So that...）
3. 验收标准（Given/When/Then 列表）
4. 非功能需求（性能/安全/兼容）
5. 范围边界（做什么/不做什么）

5 个模块，1-2 页，适合中小需求

B

完整版（适合正式项目）

1. 需求概述：背景、目标、范围
2. 用户分析：角色、场景、使用流程
3. 功能需求：功能列表 + 优先级 + 用户故事
4. 非功能需求：性能、安全、可用性、兼容性
5. 业务流程：核心流程图/状态机
6. 验收标准：每个功能的 Given/When/Then
7. 依赖与风险：外部依赖、技术风险
8. 附录：术语表、参考资料

8 个模块，适合复杂需求、跨团队协作

C

模块化（按需加载，推荐）

核心模块必填，扩展模块按需求复杂度自动判断是否需要。Agent 在对话阶段根据需求类型智能选择

🔴 必填模块：需求概述、用户故事、验收标准
🟡 条件模块：有权限需求 → 权限模型；有支付 → 支付流程；有状态变更 → 状态机
🟢 可选模块：竞品分析、数据埋点、A/B 方案
