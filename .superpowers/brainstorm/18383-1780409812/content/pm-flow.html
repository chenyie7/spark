PM Agent 的对话流控：怎么知道该结束了？

混合模式最大的难点 —— Agent 什么时候停止提问、开始写 PRD？

三阶段对话流

阶段 1
开放式探索：人类 PM 描述需求，Agent 追问理解全貌
↓ 触发条件：核心模块信息齐全
阶段 2
输出草稿 PRD：Agent 生成结构化 PRD，展示给人类 PM
↓ 触发条件：人类 PM 确认或修改完成
阶段 3
定稿 + 拆 Issues：最终 PRD + 创建 Issues + 存档

阶段 1→2 的"完成条件"怎么定义？

这是最关键的决策：Agent 怎么判断可以写草稿了？

A

信息清单检查

Agent 内部维护一个清单：目标用户✓ 核心流程✓ 边界条件✓ 权限模型✓
全部打勾 → 进入阶段 2。有缺项 → 继续追问

可控但固定，复杂需求可能漏项
B

人类主动触发

Agent 只负责问和记录，人类 PM 觉得差不多了说"出草稿吧"
Agent 才进入阶段 2。控制权完全在人手里

最简单，但人可能在不充分的时候过早触发
C

自适应检查 + 人工兜底（推荐）

Agent 根据需求类型动态生成检查清单，满足条件后自动提示"信息差不多了，要出草稿吗？"
人类可以确认，也可以继续补充

兼顾自动化和人工控制
