PM Agent 怎么和人类 PM 对话？

这是你提到的"最难的部分"——把模糊需求变成结构化 PRD 的对话过程

三种对话模式

A

问答式访谈

Agent 拿到初始需求后，按固定模板逐项提问：目标用户？核心流程？边界条件？权限模型？
一问一答，填完模板即出 PRD

优点

  • 结构化，不漏项
  • 实现简单

缺点

  • 机械化，体验差
  • 对复杂需求不够灵活
B

协作式打磨

Agent 先输出一份草稿 PRD，人类 PM 在上面批注/修改，Agent 不断迭代完善。
像两个人一起编辑文档

优点

  • 人类有掌控感
  • 适合复杂需求

缺点

  • 轮次可能很多
  • 对 Agent 理解修改意图的能力要求高
C

混合模式（先聊后写）

第一阶段：开放式对话，Agent 理解需求全貌
第二阶段：Agent 输出结构化 PRD 草稿
第三阶段：人类确认/修改，Agent 最终定稿

优点

  • 兼顾灵活和结构化
  • 最适合真实场景

缺点

  • 实现最复杂
  • 需要设计对话结束条件
