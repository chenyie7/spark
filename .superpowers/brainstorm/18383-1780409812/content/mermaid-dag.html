# Feature Map: 电商小程序
## 依赖关系图（DAG）
```mermaid graph TD F1[F1: 用户注册登录] F2[F2: 商品搜索] F3[F3: 商品详情] F4[F4: 加入购物车] F5[F5: 商品收藏] F6[F6: 下单结算]
F1 --> F2 F1 --> F3 F2 --> F3 F3 --> F4 F3 --> F5 F4 --> F6 ```
## 开发顺序（拓扑排序） 1. F1 → 2. F2 → 3. F3 → 4. F4, F5(并行) → 5. F6
---
## F1: 用户注册登录 {#F1}
**Issue:** #1   **Priority:** P0   **Depends:** 无
### Business Layer 用户可通过手机号+验证码注册/登录。首次注册自动创建档案。支持Token续期。
### Design Layer - API: `POST /api/auth/send-code`, `POST /api/auth/login` - 表: user(id, phone, nickname, created_at) - DTO: SendCodeDTO, LoginDTO / VO: LoginVO - 复用: Result<T>, SaToken 认证框架
### Implementation Layer （开发后回填） - changed_files: `AuthController.java`, `UserService.java`, `User.java` - exported_apis: `StpKit.getLoginId()`, `UserService.getById()` - exported_models: `User`, `LoginVO`
---
## F4: 加入购物车 {#F4}
**Issue:** #4   **Priority:** P1   **Depends:** F3
### Business Layer 用户从商品详情页点击"加入购物车"，系统将商品加入购物车并返回状态。同一商品再次加入数量+1。
### Design Layer - API: `POST /api/cart/items { product_id, sku_id, quantity }` - 表: cart(id, user_id), cart_item(id, cart_id, product_id, sku_id, quantity) - 复用: F1 认证拦截器、F3 的 ProductDTO、SkuDTO
### Upstream Context （自动从前序节点收集） - upstream_models: F1→User, F3→Product, Sku - upstream_apis: F1→StpKit.getLoginId(), F3→ProductService.getById()

为什么比纯 YAML 好？

📊 Mermaid DAG GitHub/GitLab 直接渲染成可视化的依赖关系图，人类一眼看懂
📝 Markdown 文本 Agent 读取 Markdown 原生解析，不需要 YAML parser。章节 + 字段清晰
🔗 锚点链接 {#F1} 让 Coder 跳转到任意节点的完整上下文，Agent 也可以用锚点定位
📦 单文件 PRD 可以和 Feature Map 合并为一个文件，或独立存放，灵活
