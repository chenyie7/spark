PM Agent = 状态同步器 + 合并决策者

PM 轮询 GitHub，把 PR 事件翻译成本地状态。Coder 读本地文件就知道该干什么。

🎯 核心设计：PM 是 GitHub → 本地的唯一桥梁

    GitHub（外部事件）                  本地 feature-map/README.md（内部状态）
    ──────────────────                 ─────────────────────────────
    Coder 创建 PR #13  ──┐
    Tester comment "✅" ──┤              F3: 🟡开发中 → 🟣待审查
    Reviewer "APPROVED" ──┤   PM 轮询    F3: 测试✅ 审查✅
                          ├──→ 读GitHub →→ F3: 🟣 → 🟢（合并！）
    Coder 修复 PR #13   ──┤              F3: 🟡（退回重来）
    Tester comment "❌"  ──┘              F3: 🟣 → 🟡（需修复）


    PM 是唯一连接两边的人。Coder/Tester/Reviewer 永远不碰 README。

🤖 PM Agent 完整职责

# ===== PM Agent ===== 每 60 秒执行一次
while not all_issues_closed:
# === 职责 1：发现新 PR → 更新状态 === prs = gh pr list --state open --json number,title,createdAt for pr in prs: fid = extract(pr.title) # "F3: 用户登录" → F3 local = read feature-map/README.md row for F{fid} if local.status == "🟡" and pr exists: # Coder 已提 PR，更新状态让 Coder 知道可以继续了 update README: F{fid} 🟡→🟣, PR=#{pr.number}
# === 职责 2：收集测试和审查结果 === for pr in prs: fid = extract(pr.title) comments = gh pr view {pr.number} --json comments reviews = gh pr view {pr.number} --json reviews
test_ok = any("🧪 ✅" in c.body for c in comments) test_fail = any("🧪 ❌" in c.body for c in comments) review_ok = any(r.state=="APPROVED" for r in reviews) review_fail = any(r.state=="CHANGES_REQUESTED" for r in reviews)
# === 职责 3：决策 === local = read README row for F{fid}
if test_ok and review_ok: gh pr merge {pr.number} --squash gh issue close {fid} update README: F{fid} 🟣→🟢 git add && git commit && git push
elif test_fail or review_fail: if local.retry >= 3: update README: F{fid} → ⚠️需人工介入 else: update README: F{fid} 🟣→🟡, retry+=1 git add && git commit && git push
else: # 还没人审查/测试完，状态保持 🟣 # 不更新 README，等待下一次轮询
sleep 60

💻 Coder 如何知道该干什么？

# ===== Coder 循环 =====
while True:
# Step 1: 读本地状态表（PM 已同步过） table = read feature-map/README.md status table
# Step 2: 检查自己上次提的 PR 是什么状态 my_last = Coder 记录的上一个 Feature ID（内存变量） if my_last and table[my_last].status == "🟣": print "等待 {my_last} 审查中..." sleep 60; continue # 不等审查完不开始下一个
if my_last and table[my_last].status == "🟡": print "{my_last} 被退回，修复中..." # 读 GitHub PR 上的失败原因 pr_num = table[my_last].pr comments = gh pr view {pr_num} --json comments,reviews read failure reasons from comments/reviews fix code → push → sleep 60; continue # 等 PM 下一轮同步
if my_last and table[my_last].status == "🟢": print "{my_last} 已通过！寻找下一个任务" my_last = None # 可以开始下一个了
# Step 3: 找下一个任务 for feature in topological_order(table): if feature.status == "🔴" and all_predecessors_are("🟢"): target = feature; break
if target is None: if all features are 🟢: print "全部完成！"; exit 0 sleep 60; continue
# Step 4: 开发 + 提 PR git checkout -b feature/F{target.id} implement feature git push && gh pr create my_last = target.id # 记住当前正在等的 Feature
# Step 5: 等待 PM 同步（下次循环检查状态） sleep 60
# Coder 只看本地 README 和 GitHub PR（读自己的失败原因） # 不写 README

🌙 完整夜间流程（所有 Agent 并行跑）

T=0min 所有 Agent 启动
T=0min PM 读 GitHub → 无 PR → 读本地 README → F1 🔴 → 不操作（等 Coder） T=0min Coder 读本地 README → F1 🔴 + 前序无 → 认领 F1 T=0min Tester 读 GitHub → 无 PR → sleep T=0min Reviewer 读 GitHub → 无 PR → sleep
T=5min Coder 完成 F1（注册）→ git push → gh pr create #11 Coder my_last=F1
T=6min PM 轮询：发现新 PR #11 "F1: 用户注册" 本地 README F1 状态 🟡 → PM 更新为 🟣（PR=#11） git commit && git push
T=6min Tester 轮询：发现 PR #11，无🧪评论 → 生成测试 → mvn test → ✅ gh pr comment #11 "🧪 ✅ 5/5 passed" T=6min Reviewer 轮询：发现 PR #11，无审查 → 4维审查 → ✅ gh pr review #11 --approve
T=7min PM 轮询：PR #11 test✅ review✅ → gh pr merge #11 --squash → gh issue close 1 → 更新 README: F1 🟣→🟢
T=7min Coder 读本地 README → F1 🟢（我上次的 PR 通过了！） → F2 前序(F1)🟢 → 认领 F2（邮箱验证）→ 开发...
... 循环到全部 Feature 🟢 ...
T=60min 全部 10 个 Feature ✅。PM 输出报告。所有 Agent 退出。

🔑 解决你的两个问题

问题怎么解决
PM 如何判断 Coder 是否完成？ PM 轮询 gh pr list。出现新 PR → Coder 已提交。
同时读本地 README：F{n} 还是 🔴 且没有对应 PR → Coder 还没开始。
F{n} 是 🟡 → Coder 已认领正在写（没有 PR）。
F{n} 是 🟣 → Coder 已提 PR，等待审查。
Coder 如何知道 PR 是否通过？ Coder 记下自己提了哪个 Feature（内存变量 my_last）。
每次循环读本地 README 的该 Feature 状态：
🟣 → 还在等，sleep 后重读
🟡 → 被退回，读 GitHub PR 找失败原因，修复
🟢 → 通过！找下一个 🔴 任务
Coder 不开始下一个 Feature，直到上一个 PR 🟢
PM 如何判断 Tester/Reviewer 是否完成？ PM 读 GitHub PR 的 comments 和 reviews。
有 "🧪" 开头的 comment → Tester 已完成。
有 APPROVED 或 CHANGES_REQUESTED → Reviewer 已完成。
否则 → 还在等，不更新 README。
