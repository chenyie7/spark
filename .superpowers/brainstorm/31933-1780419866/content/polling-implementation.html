Agent 轮询实现方案：适配不同工具

核心思路：Agent 本身不需要支持轮询——用一个极简的 Supervisor 脚本包在外面

🎯 核心问题

Agent 工具有内置轮询？怎么跑循环
Claude Code✅ CronCreate + ScheduleWakeup内置，直接调度 prompt
OpenClaw❌需要外部脚本
Cursor Agent❌需要外部脚本
GitHub Copilot CLI❌需要外部脚本
一般的 OpenAI API 调用❌需要外部脚本

🔧 方案 1：Supervisor Shell 脚本（最通用，推荐）

适用于任何 Agent 工具。一个 30 行 bash 脚本做循环，Agent CLI 每次被调用做一件事。

#!/bin/bash
# supervisor.sh — 夜间自动开发流水线 Supervisor
# 用法: ./supervisor.sh 启动后一直跑到所有 Feature 完成


echo "🌙 启动 Agent 流水线..."


# 同时启动 3 个后台进程


# 1. Coder 循环
(
  while true; do
    # 给 Agent 发一条 prompt，它完成一个任务后退出
    claude -p "读取 feature-map/README.md，
      找到前序全🟢的第一个🔴Feature，
      按 agents/coder/README.md 规范开发，
      完成后 git push 并 gh pr create。
      如果你上一个 PR 的 Feature 还是 🟣，就等待。
      如果上一个 PR 的 Feature 变成了 🟢，就找下一个。
      如果上一个 PR 的 Feature 变成了 🟡，就修复。
      完成一个动作后退出。"


    sleep 60  # 如果没活干，等 1 分钟再试
  done
) &


# 2. Tester 循环
(
  while true; do
    claude -p "读取 GitHub 上所有 open PR，
      找到还没有 🧪 Test Result 评论的 PR，
      读取 PRD.md 中对应 Feature 的验收标准，
      生成验收测试并执行 mvn test，
      将结果以 '🧪 ✅' 或 '🧪 ❌' 格式评论在 PR 上，
      完成后退出。"


    sleep 60
  done
) &


# 3. Reviewer 循环
(
  while true; do
    claude -p "读取 GitHub 上所有 open PR，
      找到还没有你的 review 的 PR，
      按 agents/reviewer/ 规范 4 维审查，
      通过则 gh pr review --approve，
      不通过则 gh pr review --request-changes，
      完成后退出。"


    sleep 60
  done
) &


# 4. PM 循环（最前面启动，最后退出）
while true; do
  claude -p "读取 feature-map/README.md 和 GitHub 上所有 open PR。
    对每个 🟣Feature：检查 PR 是否有 🧪✅ + review APPROVED → merge + close issue + 更新 README 为 🟢。
    检查 PR 是否有 ❌ → 更新 README 为 🟡（退回）。
    如果全部 Feature 都是 🟢，输出报告并创建文件 .status/ALL_DONE。"


  if [ -f .status/ALL_DONE ]; then
    echo "🌅 全部完成！"
    kill 0  # 杀掉所有后台进程
    exit 0
  fi
  sleep 60
done

🔧 方案 2：GitHub Actions 事件驱动（不用轮询）

利用 GitHub 原生的事件机制，PR 创建/更新时自动触发 Agent。

# .github/workflows/agent-pipeline.yml


name: Agent Pipeline


on:
  pull_request:
    types: [opened, synchronize]  # PR 创建或更新时触发


jobs:
  tester:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Run Tester Agent
        run: |
          claude -p "读取 PRD.md 中对应 Feature 的验收标准，生成并执行验收测试，评论 PR"


  reviewer:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Run Reviewer Agent
        run: |
          claude -p "按 agents/reviewer/ 规范 4 维审查，approve 或 request-changes"


# PM Agent 用 schedule 事件（每 5 分钟跑一次）
  pm-merge:
    runs-on: ubuntu-latest
    if: github.event_name == 'schedule'
    steps:
      - name: Run PM Agent
        run: |
          claude -p "检查 test✅+review✅ → merge → 更新 README
            全部完成 → 创建 .status/ALL_DONE 信号文件"

🔧 方案 3：Claude Code CronCreate（零脚本，最简单）

只用 Claude Code 内置的 Cron 功能，不用写一行 shell。

# 在 Claude Code 中一次性设置 4 个 cron 任务：


# 1. PM Agent — 每分钟检查
/cron "*/1 * * * *" "你是 PM Agent。读取 feature-map/README.md 和 gh pr list。
  对每个 🟣Feature：test✅+review✅ → merge+close issue+更新状态为 🟢。
  任一 ❌ → 更新状态为 🟡。全🟢 → 输出报告并停止所有 cron。"


# 2. Coder Agent — 每 5 分钟检查
/cron "*/5 * * * *" "你是 Coder Agent。读取 feature-map/README.md。
  找第一个 🔴且前序全🟢 的 Feature → 开发 → 提 PR。
  如果上一个 PR 还是 🟣 → 等待。🟡 → 修复。
  全🟢 → 退出。"


# 3. Tester Agent — 每 2 分钟检查
/cron "*/2 * * * *" "你是 Tester Agent。gh pr list 找没有 🧪 评论的 PR。
  生成验收测试 → mvn test → 评论结果。"


# 4. Reviewer Agent — 每 2 分钟检查
/cron "*/2 * * * *" "你是 Reviewer Agent。gh pr list 找没有 review 的 PR。
  4 维审查 → approve 或 request-changes。"

需要写代码 30行bash YAML workflow 文件 0行代码，纯 prompt
能否无人值守 ✅ 开 screen/tmux 即可 ✅ 完全托管 ⚠️ 需 Claude Code 保持运行
延迟 60s可调 事件触发，近乎实时 1-5分钟可调
Gitea兼容 ✅ 完全兼容 ❌ 需 Gitea Actions ✅ 完全兼容
并行能力 ✅ 4 个&后台进程并行 ✅ 独立 job 并行 ✅ 4 个 cron 并行
Agent崩溃恢复 while true 自动重试 下次事件自动触发 下次 cron 自动触发

🎯 推荐策略

晚上无人值守 → 方案 1（Supervisor 脚本）

开一个 screen 或 tmux 会话，运行 ./supervisor.sh。Agent 工具可以是 Claude Code、OpenClaw 或任何 CLI。脚本里 while true 保证崩溃也能自动恢复。第二天早上 screen -r 看日志。

最通用、最可靠。建议作为主要方案。

只用 Claude Code → 方案 3（CronCreate）

零代码。在开始睡觉前运行 4 个 /cron 命令即可。但需要电脑不锁屏、Claude Code 不退出。

最简便，但有环境限制。

有 CI 服务器（GitHub Actions / Gitea Actions） → 方案 2

最正规。PR 事件触发 Tester 和 Reviewer（不用轮询），PM 用 schedule 每 5 分钟检查一次。完全托管，不怕电脑关机。

生产级方案，需要 CI 基础设施。

Supervisor 脚本不需要考虑 Agent 工具差异

# 不指定 Agent 工具，任何 CLI 都能接入 # 只需修改 AGENT_CMD 这一个变量
# Claude Code: AGENT_CMD="claude -p"
# OpenClaw: # AGENT_CMD="openclaw run"
# 通用 OpenAI API: # AGENT_CMD="python run_agent.py"
# Cursor CLI: # AGENT_CMD="cursor-cli"
$AGENT_CMD "你的 prompt"
