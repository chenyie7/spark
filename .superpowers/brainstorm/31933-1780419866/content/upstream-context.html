Upstream Context 三个方案

目标：下游 Coder 拿到前序产物，但不需要读大量文件

方案 A：只读 Impl Layer 段落（不改任何文件）

# Coder 开发 F9 时： for dep in F3 F6 F7 F8: exports += sed -n '/## Implementation Layer/,/^## /p' feature-map/{dep}.md
# 只提取 Impl Layer 段落，不读整个文件 # 结果： F3 → LoginVO, StpKit.getLoginId() F6 → RateLimiter, LoginAttemptService F7 → Product, Sku, ProductService.getById() F8 → Cart, CartItem, CartService.addItem()

优点

  • 不改文件格式，零维护成本
  • 永远是最新的（读到的是源码）

缺点

  • Coder 每次都要执行这个聚合逻辑
  • 依赖 Feature 文件格式稳定

方案 B：自动生成 .exports 摘要文件（每个 Feature 一行）

feature-map/ ├── README.md ├── .exports ← 自动生成，每行一个 Feature 的导出 │ F1: User, UserService.create() │ F2: EmailVerificationService.isVerified() │ F3: LoginVO, StpKit.getLoginId(), AuthController │ F4: PasswordResetService │ ... ├── F1-用户注册.md └── ...
# Coder 开发 F9 时： cat feature-map/.exports | grep "^F3:\|^F6:\|^F7:\|^F8:" → 4 行，一目了然

优点

  • 读一次就拿到所有上游上下文
  • 极简格式，不会膨胀

缺点

  • .exports 需要有人维护（谁写？）
  • 数据冗余（Impl Layer 和 .exports 各一份）

方案 C：Feature 文件顶部加 exports 字段（类 frontmatter）

# F8: 加入购物车 exports: Cart, CartItem, CartService.addItem(), CartService.getCart()
Issue: #8 | Priority: P1 | Depends On: F7 | Blocks: F9
## 依赖关系（局部视图） ...
# Coder 开发 F9 时，只需读每个上游文件的第 2 行： head -2 feature-map/F3-用户注册.md head -2 feature-map/F8-加入购物车.md ...

优点

  • 不增新文件，数据跟着 Feature 文件走
  • 一行就能拿到导出清单

缺点

  • 改文件格式，需要更新 PM Agent 的模板
  • exports 行可能很长（需要折行规范）
